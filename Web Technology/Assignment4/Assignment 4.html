Summary

Our website for a webshop for mobile phones is in the end very bare bones and simple, so evaluating the accessibility and usability is quite simple. For accessibility, the website has only a few things that could break the accessibility for users. The website consists mostly of text, with only a few interactive elements. For the layout and color contrast, we tried to make it as contrasting and simple as possible, to allow for greater accessibility. For the interactive parts, they can be considered accessible to most since they conform to how similar items work on different websites. What we could have improved upon however was making it clearer what the form was used for, and what inputs were required. Another thing in our website is the sorting of the tables. If the user does not know that the tables can be sorted, there is nothing indicating that they can be sorted, this is something that could be improved upon.

Guidelines

For our site, I would pick a few relevant ones to be use of color and contrast, since we picked the colors we use on purpose to keep it both easy on the eyes and readable, keyboard accessibility, since you can navigate and use the site with a keyboard all the way, and link purpose, as we define clearly what every link does and where it takes you.

Improvement plan

Based on the Appendices A and B, the improvement plan is as follows:
- Set img tags in css
- Indent css
- Hotlink to JS file instead of having it downloaded.

2.1

Keyboard accessibility

We have two distinct functions in our website that require user input: sorting the tables, and the form. The form works with just a keyboard with no issues, since you have to type in the data anyway, you can change fields with the tab key, and post with enter. Sorting the tables however requires use of a mouse, since you can't select the table headers through a keyboard to activate them. The page does not fulfill this criterion.

Keyboard trap

If you have selected/highlighted a text field in the form, you can't move away from that part of the website with just the arrow keys, but you can move away with the tab key, which fulfills the criterion.

2.4

Our website is small and simple, with no functionality requiring any previous functionality - no central part is more important than any other. That being said, we start with an introduction, continue on with the main content, and finish with miscellaneous information in the bottom of the page. This criterion is also met.

Link purpose

All our links state where it will take the user. This criterion is met.
