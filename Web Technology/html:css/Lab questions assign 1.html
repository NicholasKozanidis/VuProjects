Question 3

Acquaint yourself with principles of color contrast, e.g. through the Color Contrast Check (Links to an external site.)Links to an external site.. Test three color combinations and write a short report of the results and how these should be interpreted.

Color contrast is an accessibility issue. Colors that are not contrasting enough can cause problems for visually impaired people to tell the content apart from the background. Colors should have a significant enough difference in both brightness and color to be accessible. Comparing a white background and a white text gives clearly the best results, as black and white are significantly different in both brightness and in hue. Dark blue and bright green result in a significant enough brightness contrast so that it would look good on a black and white screen, but fall short on the color contrast, so that it might cause issues for a visually impaired person. Bright green and bright pink together result in a significant color difference, but a very low brightness difference. So someone who has trouble telling colors apart should be able to read it, but it would not look good on a black and white monitor.

Question 4

Native apps are apps that are designed and installed on a specific device and/or operating system. Web apps are apps that work over the web without being installed on a device, and is usually accessed through a web browser. Native apps can access more of the device's power it is working on, and can perform faster and better. Web apps on the other hand are compatible with far more devices as they can be accessed over the web regardless of the architecture.

Question 5

Summarize the key multilingual issues that W3C is addressing within the "Internationalization" activity.

The multilingual issues in internationalization are all about how you write and read differently in different languages. The first issue is different characters. For instance greek, russian, many asian languages etc. use different alphabet than the latin alphabet, so there needs to be support for multiple character sets. The second issue is that in some languages you read and write in a different direction than in english, like right to left, or vertically. International apps and websites need to be built in such a way that if you mix languages written in different ways, they do not interfere with each other. There is also the issue that some languages have significantly different sentence structure, text needs to be made such that if it is translated, the length and structure of the text does not change dramatically. Lastly, culture-specific symbols can cause an issue. Symbols in one culture can mean something completely different in another culture, so when translating text, one must also take care to translate symbols.

Look up information about the new protocol for IP addresses (IPv6, IP version 6). Describe the main features of this protocol and indicate what the reasons are to move from version 4 to version 6. (Expected length: 1/4 page)

IPv6 uses 128 bits instead of 32 bits for it’s adresses, effectively creating an impossibly large number of available adresses. The switch from IPv4 to IPv6 is largely invisible to most computer users.

Differences between GET and POST: 1) Reload button: Harmless for GET but for POST, data will be re-submitted 2) Bookmark: Possible for GET, not for POST 3) Cached: Possible for GET, not for POST 4) Encoding type: for GET its application/x-www-form-urlencoded, for POST its application/x-www-form-urlencoded or multipart/form-data. Use multipart encoding for binary data 5) History: GET parameters remain in browser history, whereas not saved for POST 6) Restrictions on data length: Because the GET method adds data to the URL, that limits the data length as the URL max length is 2048 characters. No restrictions for POST 7) Restrictions on data type: Only ASCII characters allowed for GET, No restrictions for POST 8) Security: GET is less secure than POST, because the data sent is part of the URL, shouldn’t be used for passwords or other sensitive information. POST data are not stored in browser history or web server logs 9) Visibility: Data is visible via URL for GET, not visible for POST Source
