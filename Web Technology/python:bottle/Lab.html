Acquaint yourself with principles of color contrast, e.g. through the Color Contrast Check (Links to an external site.)Links to an external site.. Test three color combinations and write a short report of the results and how these should be interpreted.

Color contrast is an accessibility issue. Colors that are not contrasting enough can cause problems for visually impaired people to tell the content apart from the background. Colors should have a significant enough difference in both brightness and color to be accessible. Comparing a white background and a white text gives clearly the best results, as black and white are significantly different in both brightness and in hue. Dark blue and bright green result in a significant enough brightness contrast so that it would look good on a black and white screen, but fall short on the color contrast, so that it might cause issues for a visually impaired person. So someone who has trouble telling colors apart should be able to read it, but it would not look good on a black and white monitor.

Summarize the key multilingual issues that W3C is addressing within the "Internationalization" activity.

The multilingual issues in internationalization are all about how you write and read differently in different languages. The first issue is different characters. For instance greek, russian, many asian languages etc. use different alphabet than the latin alphabet, so there needs to be support for multiple character sets. Make sure you have read the blog post What Is A Web Server? as described in the Week 2: Reader on Blackboard. Knupp states that the GET method is the most common HTTP method. He does not discuss that the HTTP GET method is assumed to be idempotent. Explain:
  1. what idempotent means in this context
  2. how idempotent requests simplify the implementation of a caching HTTP proxy,
  3. how so-called "visitor counters" often break this assumption, and
  4. how such counters might be negatively affected by proxies.

An idempotent HTTP method can be called many times having the same outcome. No matter how many times the method is called, the result should be the same. Because these methods always have the same outcome and don't change the resource on the server side, they are easier to be cached and prefetched without any repercussions or side-effects to the resource. Visitor counters alter server data indicating how many times a user has visited a page. Meaning that the outcome of an idempotent "GET" request, will not always be the same. However, some argue that from a client's point of view, a GET request is still safe and it is up to the server to decide wether to view counters as side-effects or of simple semantic meaning.These counters or "cookies" can be affected by proxy users. In such a case, a user connects to a proxy server that filters requests but also changes the user's ip address, effectively causing faulted results to a counter. sources: Source1 Source2

Question 2

HTTP is a so-called stateless protocol. Explain what this means. Name at least one advantage and disadvantage of the fact that HTTP is stateless.

HTTP is stateless, meaning there is no link between two requests being successively carried out on the same connection. A drawback from this is that a user's state sometimes needs to be stored, as in for example e-shop's and the add to basket functionality. While the core of HTTP itself is stateless, HTTP cookies allow the use of stateful sessions, allowing session creation on each HTTP request to share the same context, or the same state. A main advantage of the statelessness of HTTP requests is that: it makes server design much simpler, because there is no need to dynamically allocate storage to deal with server-client conversations in progress. sources: Source1 Source2

Question 4

Explain the role of data.gov, in the discussion about a more transparent government. What could be a potential danger of sites like data.gov? (if you prefer, you can instead answer the same questions about the Dutch data.overheid.nl

Open data is a key ingredient to a more transparent government. It is difficult to conceal something if the facts are there for all to see. Open data empowers communities: the truth about crime rates, educational achievement, social services and so on is laid bare. Open data even drives economic growth: more small companies are springing up that extract useful information from data. Open data may even lead to more accurate conclusions and better decisions, as a wider variety of interested parties have the opportunity to examine the facts. In addition, with open data, people can explore the impact of policies on a wider range of indicators. However, open data raises some concerns, mainly in regards to privacy. Although government officials promise no leakage of personal information, this technology, as any other, is subject to misuse and abuse. Link: Source

Question 8

Explain how the Common Log Format helps developing Web log analysis software. Briefly describe a scenario in which such software is used in an experiment aimed at improving a web site's link structure. Hint: after changes to the web site have been made, one could again collect log information and compare the logs collected before the change with those collected after the change, to see if the change was effective. What would be the dependent and independent variables in this experiment? What would be your hypothesis?

The Common Log Format,is a standardized text file format used by web servers when generating server log files. Because the format is standardized, the files can be readily analyzed by a variety of web analysis programs. Each line in a file stored in the Common Log Format has the following syntax: host ident authuser date request status bytes Because these logs in essence record what happened by whom on the system, they can be useful for accounting purposes as well as others. In the website link structure example, the independent variable is the link structure itself and the dependent one is the the amount of users/Ip's sent http requests to each link. Depending on the traffic the before and after logs have recorded, we could make a fair assumption on how to redesign the structure based on user activity.

Question Bonus

In the summer of 2011, Web users in Iran became the victims of a man in the middle attack when trying to use Google's website. Explain what a man-in-the-middle-attack means in this particular scenario. The man-in-the-middle-attack became possible after a security breach in the servers of a small Dutch company called DigiNotar. Explain how it is possible that a small company in the Netherlands can break the security of Iranians using Google. Explain how the browser vendors reacted on this security breach, and how this reaction affected many Dutch government web sites.

In a man-in-the-middle attack, the attacker controls and alters the communication between two parties, without them knowing. In this particular event, the attackers used forged Google SSL certificates to spy on Iranian users of the domain. DigiNotar was a certificate authority dutch company, these certificates are responsible for verifying a digital entity's identity on the Internet and is an essential part of secure communication and public key infrastructure. However, DigiNotar's out of date software,it's poorly segmented network as well as overall faulty security system, lead to them being hacked and false certificates were issued. Those were then used to specifically target Iranian users. Fox-IT investigated the matter by reviewing the hacking tools used along with the custom scripts that were made for this very purpose as well as the log files of the activity. This lead to trust in all certificates issued by DigiNotar to be revoked by many browser and operating system developers. The Dutch government initially supported the company but quickly changed it's stance and ditched DigiNotar.
